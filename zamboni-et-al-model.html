Description of the late decision-related bias model¶

Components¶

The has three main components:

  1. the sensory representation of the visual stimulus which is carried through all the way to a decision stage. In the code below, this is labelled S. The sensory representation is modelled as a Gaussian distribution (in direction space); the variance for this (the main parameter of interest) can be obtained by fitting a cumulative Gaussian distribution function to the data from the discrimination task. Alternatively, it can be included as an additional parameter in the model fitting procedure. The distribution is centred on the direction of motion of the visual stimulus.

  2. a weighting profile, W, that applies repulsive shifts from applied to the sensory representation, S. In our implementation, the weighting function represents a late, decision-related bias. The specific shape of the function (and the fact that it is bimodal) captures both a repulsion away from the reference line, as well as the (attractive) influence of the boundary imposed by the stimulus range. Possible choices for such a function are the Gamma distribution or any other parametric curve that has a similar shape. The asymmetric nature of the Gamma function n particular (steeper close to the reference) reflects the fact that the boundary of the stimulus range, which the subject learns over time, is noisier.

  3. information about whether a reference is present or absent at the time of estimtation. If yes, then the weighting can be applied as the reference repulsion can be anchored to a particular direction. If no, then an (on average) uniform weighting is applied. This can be thought of as either no weighting or a uniform random weighting over many trials.

The inputs to the model are, for a number of trials, the stimulus direction, the subject's estimate, and whether the reference was present, absent or shifted.

The weighting function is modelled as a Gamma probability density function defined by the free parameters shape (A) and scale (B). In addition, a parameter δ describes any change in the location of the reference line from the discrimination to the estimation task phase. Also, the weighting (repulsive shift) is only applied if a reference is present at the estimation stage. These additional parameters are necessary because in our model, the repulsive shifts apply only at a late, decision-related stage and only if a reference was present.

An alternative weighting function (which has a similar shape to the Gamma function) is a difference of gaussians. Using this also provides a reasonable fit to the data.

Step through the notebook to run the model fitting for one particular data set. Only small modifications are needed to run the model fitting for the complete data set automatically.

Each row contains a trial. For present|absent data, there are 3 columns

  1. (true) stimulus direction
  2. estimated direction
  3. coherence levels:

For the data obtained in the "shifted" condition, there are 4 columns

  1. (true) stimulus direction
  2. estimated direction
  3. shift (-6º, 0º, +6º)
  4. coherence levels

The model¶

Let's build up the model by implementing one component after another.

  1. Sensory representation, S, veridical stimulus information.
  2. weighting function, W, repulsive effect (w/ given profile, if reference is present, constant otherwise)
  3. biased response (W .* S). This model also makes predictions for "incorrect trials", i.e. trials in which the subject's response is biased away from veridical, but with the opposite sign.

"""
Make the sensory representation for many different true directions at the same time.

This version of the model implements the shortcut of using the distributions of mean internal 
representationsnTrials, rather than using randomly sampled trials.

"""
function getSensoryRepresentation(trueDirections, d, σ_PP=20, nTrials=1)
    
    i = 0; # a counter
    S = zeros(length(d),length(trueDirections),nTrials)
    for μ in trueDirections
        i = i+1;
        # set up the distribution for this set of trials "thisDistribution"
        thisDistribution = Normal(μ, σ_PP ); 
        if nTrials == 1
            # then just calculate the mean internal representation
            S[:,i] = pdf(thisDistribution, d);
            
        elseif nTrials > 1
            # then just calculate the internal representation for nTrials different trials
            for iTrial = 1:nTrials
                thisMu = rand(thisDistribution, 1);
                S[:,i,iTrial] = pdf(Normal(thisMu[1], σ_PP), d);
            end
        end 
    end
    return S
    
end

# set up a function for the weighting
"calculate the weighting function (Gamma, or DOG)"
function getWeighting(p, d, kind="gamma")
    
    if (lowercase(kind) == "gamma") || (lowercase(kind) == "gamma+shift") 
        # gamma or gamma+shift
        # use a gamma function, which has to parameters.
        # the gamma function is only defined for d > 0
        # so we can flip the axis and create the other lobe
        Wpos = pdf(Gamma(p[1],p[2] ), d);
        Wneg = pdf(Gamma(p[1],p[2] ), -d);

        
    elseif lowercase(kind) == "dog"
        # use a difference of gaussians
        Wpos = max(pdf(Normal(+1,20 ), d) - pdf(Normal(-1,20 ), d), 0.0);
        Wneg = max(pdf(Normal(-1,20 ), d) - pdf(Normal(+1,20 ), d), 0.0);
        
        
    elseif lowercase(kind) == "uniform"
        # and uniform profile.
        Wpos = max(sign(d),0);
        Wneg = max(sign(-d),0);
    end
    
    # in this implementation of the model, we want both lobes.
    W = Wpos + Wneg;
    scaleFac = maximum(W);
    
    return (W./scaleFac, Wpos./scaleFac, Wneg./scaleFac)
end

"calculate the model for a given set of stimulus directions and weighting function parameters, p "
function modelFunction(trueDirections, p, weightingFunction)  

    # get the weighting function
    (W, Wpos, Wneg) = getWeighting(p,d, weightingFunction);
    
    # get the sensory representation
    S = getSensoryRepresentation(trueDirections, d, p[end], 1);
    S = squeeze(S,3);
    
    i = 0;
    modelFit = zeros(length(trueDirections),2);
    WSRpos = zeros(S);
    WSRneg = zeros(S);
    
    # loop through each "column" of the dataset (for each stimulus direction...)
    for μ in trueDirections
        i = i+1;
        WSRpos[:,i] = Wpos .* S[:,i];
        WSRneg[:,i] = Wneg .* S[:,i];
        
        idxPos = indmax(WSRpos[:,i]); ; # find max of element-wise multiply 
        idxNeg = indmax(WSRneg[:,i]); ; # find max of element-wise multiply 
        
        modelFit[i,1] = d[idxPos]
        modelFit[i,2] = d[idxNeg]
    end
    
    if weightingFunction == "gamma+shift"
        # then we also need to deal with the shift parameter for the prediction
        modelFit = modelFit .+ p[3];
    end
    
    WSR = WSRpos + WSRneg;
    
    return ( modelFit, WSR, WSRpos, WSRneg)
end

"Fitting function needs to only take x,p and return modelfit values"
function fittingFunction(tt, p)
    v,w,s = modelFunction(tt, p, weightingFunction)  ;
    return v
end

function fitPlot(x, y)

# a function that returns the error between the data and the model
# closure: perceivedDirections is already present

" Calculate the sum of squared errors between data points and the model prediction"
function sqerror(params)   
    cumulativeErr = 0.0
    thisErr = 0.0
    for ii in 1:length(trueDirectionsCenters)
        pred_i = fittingFunction(trueDirectionsCenters[ii], params);
        # there are two predictions (one for estimates reported CCW to reference: -ve, another +ve)
        thisErr = (perceivedDirections[ii] - pred_i).^2;
        # take the smaller error (the prediction of the two that is closer to the data point)
        cumulativeErr += minimum(thisErr);
    end
    return cumulativeErr 
end
